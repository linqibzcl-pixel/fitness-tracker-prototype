# Squat Counter Prototype

## 项目简介
一个基于 AI 工具快速搭建的深蹲计数器原型，用于验证健身工具的页面结构、训练反馈和数据展示方式。

## 核心功能
1.多动作运动计数/计时
-支持深蹲、俯卧撑、仰卧起坐、平板支撑四类动作
-次数型动作与计时型动作共用同一训练面板，切换动作后图标、单位、目标值、计数规则同步变化
2.训练过程中的即时数据展示
-支持目标值切换、训练时长显示、卡路里动态估算
-体重输入由普通输入框优化为滑动条，提升交互体验
3.运动记录与日历查看
-展示过去一个月的训练记录日历图表
-支持月份切换、按动作筛选记录、点击某一天查看当日训练详情
4.AI辅助原型迭代与本地运行验证
-基于 bolt.new 生成初始原型，并在 Cursor 中持续通过 prompt 迭代功能
-完成本地环境配置、依赖安装、运行验证和页面效果检查

## 当前页面包含哪些模块
1.顶部动作选择区
支持深蹲、俯卧撑、仰卧起坐、平板支撑切换
2.训练主面板
包含动作展示、计数/计时、目标值切换、开始/控制按钮、训练时长
3.卡路里估算与体重设置区
支持体重滑条输入，估算当前训练消耗
4.运动记录日历与详情区
-展示过去一个月训练记录
-支持月份切换、按动作筛选、点击某天查看详情

## 改动方案与顺序
第一轮：先做动作切换器 + 2 到 3 个动作支持【完成了】
第二轮：
1.让不同动作对应不同图标、标题和卡路里系数（图标、动作名称、单位（次 / 秒）、训练目标值、卡路里公式、历史记录筛选‘日期、动作类型、完成次数/时长、组数、估算消耗’）【完成了】
2.点击某一天看详情（日历格子仍然显示总量，但点击某天后弹出详情：深蹲：30 次、俯卧撑：15 次、平板支撑：60 秒、总卡路里：xx kcal）【完成了】
3.训练完成后写入历史（比如一条记录包含：时间、动作类型、完成值（次/秒）、目标值、是否完成、本次消耗、体重）
4.添加了模拟心率模块。心率数值应随训练时间增长在110到 160 bpm 之间随机波动，并显示对应的运动强度区间（如‘燃脂’或‘有氧’）。
这样后面：日历、历史、周报、统计页都会更好做）
第三轮：再做历史记录按动作分类
第四轮：最后再做循环训练模式（多个动作按顺序完成）

## 难点与解决思路
1.项目里比较难的点：
从单一深蹲计数器扩展到多动作统一面板
不同动作涉及不同图标、单位、目标值、卡路里公式，平板支撑还涉及计时逻辑，不能简单复制页面
多动作场景下的历史与日历展示
次数型动作和计时型动作的数据表达方式不同，日历既要支持总览，也要支持按动作筛选和点击查看某日详情
2.你是怎么解决的：
用配置驱动代替写死逻辑
将动作名称、模式（次数/计时）、目标选项、估算参数等集中配置
让不同动作共用一套训练面板，只根据动作类型动态切换展示和计算规则
采用“小步迭代 + review 验收”的方式推进
不一次性做很大改动，而是先完成动作切换，再做图标/规则联动，再做日历详情
每次通过 Cursor Review 检查改动文件和页面效果，避免 AI 重构过度或改偏方向
